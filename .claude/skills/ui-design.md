# UI Design Skill

修改和优化前端UI设计的工作流程。

## 适用场景

- 调整页面布局和间距
- 修改颜色、字体、图标等视觉元素
- 添加或优化交互动效（悬停、点击、过渡动画）
- 响应式布局调整
- 新增UI组件或修改现有组件样式
- 修复UI相关的视觉问题（对齐、溢出、空白等）

## 工作流程

### 1. 理解需求

- 确认用户想要的视觉效果
- 如果描述模糊，提供 2-3 个方案让用户选择
- 明确修改范围（全局/局部，哪些页面/组件）

### 2. 检查现有代码

- **先读后改**：必须先读取目标文件，理解现有结构和样式
- 检查是否有重复的 CSS 定义（常见问题：同一选择符出现多次，后面的覆盖前面的）
- 检查 HTML 结构是否与 CSS 选择器匹配
- 检查 JS 中是否有动态样式操作影响目标元素

### 3. 实现修改

#### CSS 修改原则

- **避免过度缩小**：字体和元素不要小于可读尺寸（最小 11px 字体，24px 可点击区域）
- **间距渐进调整**：每次调整幅度不超过 4px，避免从过大直接跳到过小
- **检查重复定义**：修改前先搜索是否有同名选择器已定义，避免冲突
- **使用 CSS 变量**：如果项目已有 CSS 变量系统，优先使用
- **保持一致性**：新样式应与项目现有风格一致

#### HTML 修改原则

- **保持语义化**：使用合适的 HTML 标签
- **避免内联样式**：除非是动态计算的值
- **检查 ID 唯一性**：确保新增的 ID 不与现有冲突
- **XSS 防护**：用户输入内容使用 `textContent` 而非 `innerHTML`

#### JS 修改原则

- **事件绑定**：优先使用 `addEventListener` 而非 `onclick` 属性
- **防抖/节流**：连续触发的事件（scroll、resize、wheel）必须添加防抖
- **悬停面板**：使用 `mouseenter/mouseleave` + 延迟关闭（100-200ms），避免鼠标移动到面板时关闭
- **状态管理**：UI 状态变化时同步更新 DOM 和数据

### 4. 验证

- **必须启动应用验证**：UI 修改后必须启动应用在浏览器中查看效果
- **硬刷新**：提醒用户清除浏览器缓存（Ctrl+Shift+R / Cmd+Shift+R）
- **检查多个场景**：
  - 不同内容长度（长文本、短文本、空内容）
  - 不同屏幕尺寸
  - 交互状态（悬停、点击、激活）
- **检查副作用**：确认修改没有影响其他组件

### 5. 提交

- 提交信息格式：`ui: 简要描述修改内容`
- 不要推送，除非用户明确要求

## 常见问题排查

### 样式不生效

1. **浏览器缓存**：最常见原因，硬刷新解决
2. **Flask 模板缓存**：设置 `TEMPLATES_AUTO_RELOAD = True` 和 `SEND_FILE_MAX_AGE_DEFAULT = 0`
3. **CSS 优先级**：检查是否有更高优先级的选择器覆盖
4. **重复定义**：搜索同名选择器，后面的会覆盖前面的

### 布局异常

1. **多余空白**：检查是否有 `margin`、`padding`、`line-height` 导致
2. **元素溢出**：检查 `overflow` 和 `width/height` 设置
3. **Flex/Grid 问题**：检查 `flex-shrink`、`gap`、`align-items` 等
4. **绝对定位**：检查 `position: absolute` 的元素是否脱离文档流

### 交互问题

1. **面板闪退**：添加 `mouseleave` 延迟（100-200ms）
2. **点击穿透**：检查 `z-index` 和 `pointer-events`
3. **滚动冲突**：检查 `overflow` 和 `e.preventDefault()`
